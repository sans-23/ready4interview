<section className="content-section">
  <h2 id="hash-set-internals">HashSet: Backing HashMap & PRESENT Dummy</h2>
  <p>
    A `HashSet` is backed internally by a transient `HashMap` instance:
  </p>

```java
public class HashSet<E> implements Set<E> {
    private transient HashMap<E,Object> map;

    // Dummy value to associate with an Object in the backing Map
    private static final Object PRESENT = new Object();

    public HashSet() {
        map = new HashMap<>();
    }
}
```

  <h3>1. Enforcing Uniqueness</h3>
  <p>
    When you insert an element using `add(E e)`, `HashSet` calls `put()` on its backing `HashMap`, using your element as the **key** and the dummy `PRESENT` object as the **value**:
  </p>

```java
public boolean add(E e) {
    return map.put(e, PRESENT) == null;
}
```

  <p>
    If the key does not exist in the map, `map.put()` returns `null`, and the set's `add()` method returns `true`. If the key is already present, `map.put()` returns the existing value (`PRESENT`), and `add()` returns `false`, preventing duplicates.
  </p>

  <h3>2. Hash Code & Equals Dependency</h3>
  <p>
    Because elements act as keys in a `HashMap`, they must correctly implement `hashCode()` and `equals()`. 
  </p>
  <ul>
    <li>`hashCode()` determines the bucket location.</li>
    <li>`equals()` resolves collisions within the bucket.</li>
  </ul>
  <p>
    If these methods are inconsistent, the set will allow duplicate objects to be added, causing unexpected behavior.
  </p>
</section>

<section className="content-section">
  <h2 id="linked-hash-set-internals">LinkedHashSet: LinkedHashMap Ordering</h2>
  <p>
    `LinkedHashSet` extends `HashSet`. Rather than duplicating code, it calls a package-private superconstructor in `HashSet` that initializes a `LinkedHashMap`:
  </p>

```java
// HashSet package-private constructor used by LinkedHashSet
HashSet(int initialCapacity, float loadFactor, boolean dummy) {
    map = new LinkedHashMap<>(initialCapacity, loadFactor);
}
```

  <p>
    <strong>Key mechanics of LinkedHashSet:</strong>
  </p>
  <ul>
    <li>
      <strong>Insertion Order:</strong> Iterating through a `LinkedHashSet` returns elements in the exact order they were inserted. This is managed by the doubly-linked list pointers inside the backing `LinkedHashMap`.
    </li>
    <li>
      <strong>Lookup Performance:</strong> Retains the $O(1)$ performance of `HashSet` for search, insertion, and deletion.
    </li>
    <li>
      <strong>Memory Overhead:</strong> Slightly higher memory footprint than `HashSet` due to the overhead of storing linked list pointers for each entry.
    </li>
  </ul>
</section>

<section className="content-section">
  <h2 id="tree-set-internals">TreeSet: TreeMap Sorted & Bounds</h2>
  <p>
    `TreeSet` is backed by a `TreeMap` instance. It implements `NavigableSet` (which extends `SortedSet`), ordering its elements in a Red-Black Tree.
  </p>

  <h3>1. Sorted Performance</h3>
  <p>
    Unlike `HashSet`, which offers $O(1)$ average time complexity, `TreeSet` operations (`add`, `remove`, `contains`) run in **$O(\log N)$** time. This is because elements must be inserted into a balanced binary tree, which requires comparison steps.
  </p>

  <h3>2. NavigableSet API Range Bounds</h3>
  <p>
    Because the tree is sorted, `TreeSet` can perform range queries and nearest-match lookups:
  </p>
  <ul>
    <li>`first()` / `last()`: Returns the lowest/highest elements.</li>
    <li>`lower(e)` / `higher(e)`: Returns the greatest element strictly less than `e`, or the least element strictly greater than `e`.</li>
    <li>`floor(e)` / `ceiling(e)`: Returns the greatest element less than or equal to `e`, or the least element greater than or equal to `e`.</li>
    <li>`subSet(fromElement, toElement)`: Returns a view of the portion of the set ranging from `fromElement` to `toElement`.</li>
  </ul>
</section>

  <h2 id="conclusion">Conclusion & Takeaways</h2>
  <p>
    Sets in Java leverage Map key properties to enforce element uniqueness. Selecting between HashSet for raw speed, LinkedHashSet to preserve ordering, and TreeSet for sorted boundary lookups is key to choosing the correct set structure.
  </p>

      Key Interview Takeaways
    </h3>
    <ul>
      <li><strong>How does HashSet prevent duplicates?</strong> It maps elements as keys in a backing `HashMap` instance, assigning them a dummy `PRESENT` object value. `HashMap` keys are guaranteed to be unique.</li>
      <li><strong>What happens when you add a duplicate to HashSet?</strong> The call delegates to `map.put(key, PRESENT)`. Since the key already exists, the map returns `PRESENT` instead of `null`, causing `add()` to return `false` without modifying the set.</li>
      <li><strong>Why does TreeSet disallow nulls?</strong> Because tree insertions compare keys to determine parent-child placement. Comparing a null key throws a `NullPointerException`.</li>
      <li><strong>Compare set iteration performance:</strong> `LinkedHashSet` iteration is generally faster than `HashSet` because it iterates over the linked list pointers rather than checking empty capacity buckets in a hash table.</li>
    </ul>
